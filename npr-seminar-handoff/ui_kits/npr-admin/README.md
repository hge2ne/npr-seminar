# npr 입시설명회 운영 콘솔 — UI Kit

학원 npr의 대입설명회 운영 백오피스(데스크톱 1366) + 학부모 모바일 예약(390). 디자인 시스템 하이파이, 전 화면 라우팅·조작 가능. 모든 데이터는 in-memory 시드(가상)이며 문자·QR·카메라는 목업.

역할은 **단일 관리자 계정** 하나(역할 게이팅 없음). 학부모/학생은 로그인 없이 `mobile.html`만 사용.

## 화면 (허브 + 7모듈 + 모바일)

- **LoginScreen** — 단일 관리자 로그인(qr-poc식). `npr-user` localStorage 저장, 아바타 메뉴 로그아웃
- **HubScreen** — 카드 런처 7장(요약 수치) + 설명회 칩 + 모바일 예약 링크. 간담회 카드는 '준비 중'
- **StudentsScreen (예약 명단)** — 설명회 드랍다운(360px)+캠퍼스 탭, 검색·단위 탭(전체·초등·중1~3·특목·고등·과학·비재원생)·담임 필터, 11열 가운데 정렬 테이블(No·학번·학생이름·반명·학교·학년·담임명·연락처(모/부)·예약·입장·로그), 예약 드랍다운(`-`/예약(모)/(부)/(모,부)/수동 예약/예약취소), 입장·로그 2줄+이력 툴팁, 수동 추가(비재원생: 모/부 탭 + 경로 전화/선생님/현장 예약)
- **SmsScreen (문자 발송)** — 템플릿 목록/생성/저장/삭제(마지막 1개 보호), 변수 칩({학생명}{설명회명}{일시}{장소}{QR링크}{문의전화}), 바이트/LMS 카운터, 대상 = 설명회 × 그룹 4종 × 캠퍼스(전체 없음), 캠퍼스별 발신·문의번호(CAMPUS_INFO), 실기기 프리뷰, 발송 로그(자동 배지)
- **SessionsScreen (설명회 운영)** — 좌측 목록(드래그 리사이즈 190~460px)+정원 게이지, 새 설명회 생성(2열 모달 + 페이지 꾸미기: 배너 테마 3종·안내문, 시간 입력 없음 10:00 고정), 현황 스탯 4장, 만족도 설문 문자 편집({설문링크})·발송(입장>0), 설문 결과 9열 테이블(+건수·응답률·평균, 후기 툴팁, 사진 링크·엑셀 목업), 종료(미체크→내부 no_show)·삭제
- **CounselScreen (간담회 예약)** — 준비 중 placeholder(미구현)
- **StatsScreen (통계)** — 설명회 드랍다운+캠퍼스 탭, 지표 5카드(전체 인원·총 예약·입장·참석률·노쇼율), 채널 도넛(모바일/수동), 단위별 예약률·참석률 듀얼 바, 평균 만족도
- **ScannerScreen (QR 스캐너)** — 스캐너 #1~#4 모니터링(ON/OFF·배터리 40%↓ 경고), 카메라 권한 목업, 전체화면 스캔(데모) → 입장 확인 팝업 2초, 현장 입장(연락처 뒷 4자리, 모/부 모두 매칭 → 체크인 or 재원생 즉석 예약, 비재원생은 수동 추가 안내)
- **PreviewScreen (모바일 프리뷰)** — 폰 프레임(390×844 스케일) 속 MobileFlow, 테스트 전용. 프리뷰 예약·설문이 콘솔 store에 즉시 반영
- **MobileFlow / mobile.html** — 캠퍼스(STEP 1) → 설명회(STEP 2, 마감 배지) → 재원생 조회(모/부 매칭) → 자녀 다중 선택(가족, 이미 예약됨 배지로 중복 방지) + 참석 학부모 칩(모/부 복수, attendField 설명회만) → 자녀별 QR 티켓(예약변경/취소 + URL 공유) / 비재원생 폼(동일 연락처 중복 차단) / 조회(연락처만)·회차 변경·취소 / 만족도 설문(별점 필수·후기·사진)

## 구조

- `index.html` 셸: 로그인 → 허브 ↔ TopNav 7탭(라우트 `npr-route`, 로고=허브), 중앙 store(React state)로 전 화면 데이터 공유
- `mobile.html` 셸: MobileFlow + 모바일 store 서브셋(액션 시맨틱은 index.html App과 동일)
- `data.js` `window.NPR_DATA`(캠퍼스·CAMPUS_INFO·unitOf·재원생 30·단일 설명회 시드) · `icons.jsx` `window.NPRIcons` · `shared.jsx` QrBox/ResStatusBadge/StatCard/EmptyState/KV
- 디자인 시스템: `../../_ds_bundle.js`(`window.DesignSystem_179b2a`) + `../../styles.css`(토큰)
- 상태 모델: `reserved → entered / no_show(내부) / cancelled` · 로그 라벨은 행위자 기준(웹앱/수동 예약·취소) · 코드 `NPR-SSx-0000`(재발급 `-Rn`)
