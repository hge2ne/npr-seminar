# npr 대입설명회 운영 시스템 — 기능 명세서

버전 **v4.0** · 2026-07-16 · 근거: `ui_kits/npr-admin/` 실제 구현 (전 화면 코드 검증)
대상 사용자: 학원 관리자(단일 역할) / 학부모·학생(모바일 예약, 로그인 없음)

> v4.0은 v3.0~v3.3에 걸쳐 누적된 변경 노트를 본문에 **통합한 현행화 판** — 기능 변화 없음. 플로우 노드 참조는 `flows.json`(v3.3), 구현 추적용 목록은 `기능리스트.md`.

## 1. 표면 및 진입

| 표면 | 진입 | 설명 |
|---|---|---|
| 운영 콘솔 (데스크톱 1366) | `index.html` | 로그인(단일 관리자) → 카드 런처 허브 + 7개 모듈 |
| 학부모 모바일 예약 (390) | `mobile.html` | 로그인 없음 — 캠퍼스 → 설명회 → 예약 → QR 발급, 조회·변경·취소, 만족도 설문 |

### 1.1 내비게이션
- **로그인**: 단일 관리자. `npr-user`(localStorage) 저장, 아바타 메뉴에서 로그아웃. 학부모/학생은 로그인 없이 모바일만.
- **허브 = 카드 런처**: 7개 모듈 카드(예약 명단·문자 발송·설명회 운영·간담회 예약(준비 중)·통계·QR 스캐너·모바일 프리뷰) + 설명회 칩 + 모바일 예약 링크.
- **모듈 내부 = 상단 탭바(TopNav)**: 탭 7개, 로고 클릭 시 허브 복귀, 우측 알림 벨 + 아바타. 현재 라우트는 `npr-route`에 저장.

## 2. 데이터 모델 (시드: `data.js`)

- **캠퍼스** (`CAMPUSES`) + 캠퍼스별 발신·문의전화 (`CAMPUS_INFO`, 발신번호 = 문의전화):

| 캠퍼스 | 발신번호 · 문의전화 |
|---|---|
| 송파캠퍼스 | 02-413-2652 |
| 위례캠퍼스 | 02-425-2652 |
| 광진캠퍼스 | 02-422-2652 |

- **Teacher** `{ id, name }` — 담임
- **Class** `{ id, name(반명), teacherId, teacherName, campus, unit }` — 반명이 단위를 결정(2.1)
- **Student** `{ id, no(학번), name, school, grade, classId, className, teacherName, campus, unit, motherPhone, fatherPhone }` — 재원생 30명
- **Session** `{ id, title, campus(전체|특정 캠퍼스), date, round, time, place, capacity, desc, attendField(참석 인원 수집 여부), active, ended, banner, notice, surveySms }`
  - 시드는 **단일 설명회 운영**: '2026 대학교 입시 설명회' — 8/21(금) 1회차 11:00 · 송파 교통회관 2층 대강당 · 정원 800 · campus=전체 · attendField=true. 다중 설명회 코드는 유지.
- **Reservation** `{ id, code(NPR-SSx-0000), sessionId, studentId|null, name, school, grade, className, teacherName, campus, unit, phone, channel(mobile|manual), status, reservedBy(모|부|모,부|''), source(웹앱|수동|전화예약|선생님 예약|현장 예약), attendCount, member, groupId(가족), scannerNo, enteredAt, logs[]{label,at}, history[](회차 이동), codeHistory[](QR 재발급 -Rn), audit[](입장 취소) }`
- **SurveyResponse** `{ id, sessionId, campus, unit, student, className, teacherName, phone, rating(1~5), comment(후기), photo(사진첨부 여부) }`
- **SmsTemplate** `{ id, name, body }` · **SmsLog** `{ id, when, to, template, session, campus, ok, fail, auto? }`
- **Device** `{ id, label, model, scannerNo(1~4), on, battery, last }`

**시각 포맷**: `MM/DD(요일) HH:MM:SS` — 예: `07/16(수) 14:03:22` (`fmtNow`)

**store 액션**: `setRosterReservation(드랍다운 상태 전이) / addGuest(수동 추가) / addReservation / addReservationGroup(가족·모/부 매칭) / cancelReservation(by: parent→'웹앱 예약 취소', staff→'수동 예약 취소') / checkIn(id, scannerNo) / moveReservation / reissueCode / rollbackEntry / endSession(미체크→내부 no_show) / addSession / deleteSession / updateSurveySms / addSurveyResponse / saveTemplate / addTemplate / deleteTemplate / addSmsLog` — 모든 화면이 동일 store 공유(프리뷰 예약·설문이 콘솔에 즉시 반영).

### 2.1 단위 판별 규칙 (`unitOf(반명)` — 운영 엑셀 수식과 동일한 IF 순서)
1. `과초6·과고1·과고2·과고3·과예중1·과예고1·과1·과2·과3` 접두 → **과학**
2. `예고1` → 예고1 / `예1S`·`예1영` → **특목** / `예1` → 예중1
3. `초6`·`초5` → **특목** / `중1`·`중2`·`중3` → **특목**
4. `초3` → **초등** / 첫 글자 `4`·`5`·`6` → **초등**
5. `고1`·`고2`·`고3` → **고등**
6. 첫 글자 `1`/`2`/`3` → **중등1/중등2/중등3**

단위 탭: `전체·초등·중1·중2·중3·특목·고등·과학(·비재원생)` — 중N 탭=중등N, **특목 탭은 예중1·예고1 포함**. 비재원생은 단위 없음.

## 4. 예약 명단 (StudentsScreen)

| # | 기능 | 상세 |
|---|---|---|
| 4.1 | 상단 컨트롤 | 좌: 제목 '예약 명단'(인원수 표기 없음). 우: **설명회 드랍다운(한 줄 고정 360px)**, 그 아래 **캠퍼스 탭(전체/송파/위례/광진, 기본 송파캠퍼스)** |
| 4.2 | 필터 | 검색(이름·학교·연락처) + **단위 탭(전체·초등·중1·중2·중3·특목·고등·과학·비재원생)** + **담임 Select** |
| 4.3 | 명단 범위 | 선택 설명회 기준 재원생 전체 + 비재원생 |
| 4.4 | 11열 테이블 (전부 가운데 정렬) | **No(1부터) · 학번 · 학생이름 · 반명 · 학교 · 학년 · 담임명 · 학부모연락처(모/부 위아래) · 예약 · 입장 · 로그** |
| 4.5 | 예약 드랍다운 | `- / 예약 (모) / 예약 (부) / 예약 (모,부) / 수동 예약 / 예약취소` — 없으면 `-`. 예약(모/부/모,부)은 **참석 학부모 표시**일 뿐, 로그는 행위자 기준(4.7). 변경 시 store 전이 + 로그 누적. 비재원생 행은 경로가 source로 남음 |
| 4.6 | 입장 열 | 입장 시 **2줄: `#스캐너번호` ↵ `MM/DD(요일) HH:MM:SS`**, 예약만 있으면 `대기`, 없거나 취소면 `—`. 노쇼 열 없음(내부 집계만) |
| 4.7 | 로그 열 | 최근 로그 2줄(라벨 ↵ 시각) + **hover 시 전체 이력 툴팁(+N 배지)**. 라벨은 행위자 기준: 학부모=`웹앱 예약`/`웹앱 예약 취소`, 관리자 조작=`수동 예약`/`수동 예약 취소` |
| 4.8 | 수동 추가 (비재원생) | 버튼 '수동 추가'. 폼: 이름*·연락처*·학교·학년·캠퍼스 + **참석 학부모(모/부 탭, 복수 가능)** + **예약 경로(전화예약/선생님 예약/현장 예약)** — 반명 없음. 명단 반영: 반명=`비재원생`, 담임=공란, source=선택 경로, 로그 `수동 예약` |

| 6.7 | 설명회 삭제 | 확인 다이얼로그 → 세션 + 연동 예약 제거 |

## 10. 모바일 예약 (MobileFlow — 프리뷰·실기기 공용)

| # | 기능 | 상세 |
|---|---|---|
| 10.1 | 캠퍼스 선택 (STEP 1) | 송파/위례/광진 카드 — 선택 캠퍼스의 설명회만 노출(campus=전체 설명회는 모두 포함) |
| 10.2 | 설명회 선택 (STEP 2) | 마감 배지(카드 비활성)·안내문 표시. 잔여석 UI 숨김(로직 유지) |
| 10.3 | 재원생 조회 | 학부모 연락처(**모/부 모두 매칭**) → 자녀 카드 다중 선택(가족). **이미 예약된 자녀는 '이미 예약됨' 배지 + 선택 불가**(중복 예약 방지) |
| 10.4 | 참석 학부모 선택 | **모/부 칩(복수 선택 가능)** — reservedBy=모/부/모,부, attendCount=선택 수. **참석 인원 수집(attendField) 설명회에서만 노출** — 미수집 시 reservedBy=매칭된 모/부 |
| 10.5 | 예약 확정 | 가족은 자녀별 QR + groupId, 로그 `웹앱 예약` |
| 10.6 | 비재원생 예약 | 폼: 이름*·학교·학년·연락처* + 참석 학부모(모/부, attendField 시) — 선택 캠퍼스로 명단 반영. **동일 연락처 유효 예약 있으면 차단 안내** |
| 10.7 | 예약 완료 티켓 | **'예약 학생명'** 표기, 참석 학부모(모·부) 표기, 가족 목록. 하단 **예약변경/취소 버튼 + URL 공유(링크 복사)**. 예약번호·잔여석·캘린더 UI 없음 |
| 10.8 | 조회·변경·취소 | **연락처만으로 조회** → 예약 내역 목록 → 선택 → 티켓(예약번호 미표기) → 회차 변경(QR 유지, 단일 회차면 빈 목록 안내) / 취소(로그 `웹앱 예약 취소`) |
| 10.9 | 만족도 설문 | **별점(1~5, 필수) + 후기(주관식) + 사진 첨부(목업 토글)** → 제출 시 콘솔 설문 결과에 즉시 반영 |
